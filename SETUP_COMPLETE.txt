================================================================================
  DATABASE ARCHITECTURE
================================================================================

Three Tables:
   1. users (id, email, password, name, role, employeeId)
      └─ Relationships: assessments[], draftData

   2. draft_assessments (userId, basicInfo, clinicalAssessment, lifestyle, ...)
      └─ Purpose: Work-in-progress assessments (auto-save)

   3. assessments (id, userId, employeeId, basicInfo, status, submittedAt, ...)
      └─ Purpose: Final submitted records (immutable)

Data Flow:
   User Form Input → Save → /api/assessments/draft → draft_assessments table
                                    ↓
                           User edits & saves multiple times
                                    ↓
                    User clicks Submit → /api/assessments/
                                    ↓
                    Data moves to assessments table
                                    ↓
                    Draft deleted automatically

================================================================================
  COMMON ISSUES & SOLUTIONS
================================================================================

"Cannot connect to database"
   → Is PostgreSQL running?
      • Docker: docker ps | grep postgres
      • macOS: brew services list | grep postgresql
      • Linux: sudo systemctl status postgresql
   → Is DATABASE_URL correct in .env.local?
      • For Docker: postgresql://postgres:postgres@localhost:5432/health_assessment

"Prisma Client not found"
   → Run: npm install
   → Then: npx prisma generate
   → Restart: npm run dev

"Migration already exists"
   → This is fine! Migrations are idempotent
   → Run: npx prisma migrate deploy

"Tables don't exist"
   → Run: npx prisma migrate dev --name init

Full troubleshooting in: PRISMA_SETUP.md (section: Troubleshooting)
